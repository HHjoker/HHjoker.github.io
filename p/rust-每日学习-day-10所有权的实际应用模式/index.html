1. 函数中的所有权模式

在 Rust 中，函数参数可以使用所有权、不可变引用或可变引用，选择哪种方式取决于函数的需求。

模式一：获取所有权

当函数需要消耗值或进行所有权转移时：

fn eat_string(mut s: String) -> String {

// 'a 是生命周期参数
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() {
        x
    } else {
        y
    }
}

fn main() {
    let string1 = String::from("hello");
    
    {
        let string2 = String::from("world");
        
        // result 的生命周期受限于 string1 和 string2 中较短的那个
        let result = longest(&string1, &string2);
        println!("最长：{}", result);
        // string2 在这里离开作用域
    }
    // result 不能再使用，因为它可能引用了 string2
}

// 生命周期 'a 表示：返回的引用至少和两个参数活得一样久
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() {
        x
    } else {
        y
    }
}

use std::rc::Rc;

fn main() {
    // Rc 允许多个所有者
    let a = Rc::new(String::from("hello"));
    
    // clone 增加引用计数，不是深拷贝
    let b = Rc::clone(&a);
    let c = Rc::clone(&a);
    
    // 三个变量共享同一个 String
    println!("a: {}, b: {}, c: {}", a, b, c);
    
    // 引用计数
    println!("引用计数：{}", Rc::strong_count(&a)); // 输出：3
    
    // 当最后一个所有者离开作用域，内存才会被释放
    drop(b);
    println!("drop b 后计数：{}", Rc::strong_count(&a)); // 输出：2
    
    drop(c);
    println!("drop c 后计数：{}", Rc::strong_count(&a)); // 输出：1
}

use std::rc::Rc;

enum List {
    Cons(i32, Rc<List>),
    Nil,
}

fn main() {
    let a = Rc::new(List::Cons(1, Rc::new(List::Cons(2, Rc::new(List::Nil)))));
    
    // b 和 c 共享 a 的所有权
    let b = List::Cons(3, Rc::clone(&a));
    let c = List::Cons(4, Rc::clone(&a));
    
    // a 被三个地方引用
    println!("引用计数：{}", Rc::strong_count(&a)); // 输出：3
}

4. RefCell - 内部可变性

RefCell<T> 允许在只有不可变引用时修改内部数据（运行时借用检查）：

use std::cell::RefCell;

fn main() {
    let data = RefCell::new(5);
    
    // 不可变借用
    {
        let borrowed = data.borrow();
        println!("当前值：{}", *borrowed);
    } // borrowed 离开作用域，借用结束
    
    // 可变借用
    {
        let mut borrowed = data.borrow_mut();
        *borrowed = 10;
    }
    
    println!("修改后的值：{}", *data.borrow()); // 输出：10
}

use std::cell::RefCell;
use std::rc::Rc;

fn main() {
    // Rc 提供多所有权，RefCell 提供内部可变性
    let shared_data = Rc::new(RefCell::new(String::from("hello")));
    
    let a = Rc::clone(&shared_data);
    let b = Rc::clone(&shared_data);
    
    // 通过任何一个引用都可以修改数据
    a.borrow_mut().push_str(", world");
    
    println!("a: {}", a.borrow()); // 输出：hello, world
    println!("b: {}", b.borrow()); // 输出：hello, world
    
    // 所有引用看到相同的修改
}

use std::borrow::Cow;

fn add_exclamation(text: &str) -> Cow<str> {
    if text.ends_with('!') {
        // 不需要克隆，直接返回借用
        Cow::Borrowed(text)
    } else {
        // 需要修改时克隆
        Cow::Owned(format!("{}!", text))
    }
}

fn main() {
    let s1 = "hello!";
    let s2 = "hello";
    
    let result1 = add_exclamation(s1); // 借用
    let result2 = add_exclamation(s2); // 拥有
    
    println!("{}", result1); // 输出：hello!
    println!("{}", result2); // 输出：hello!
}

use std::cell::RefCell;
use std::rc::Rc;

// 任务结构
#[derive(Debug)]
struct Task {
    description: String,
    completed: bool,
}

// 待办列表，支持多所有者和内部可变性
struct TodoList {
    tasks: RefCell<Vec<Rc<Task>>>,
}

impl TodoList {
    fn new() -> Self {
        TodoList {
            tasks: RefCell::new(Vec::new()),
        }
    }
    
    fn add_task(&self, description: String) -> Rc<Task> {
        let task = Rc::new(Task {
            description,
            completed: false,
        });
        
        self.tasks.borrow_mut().push(Rc::clone(&task));
        task
    }
    
    fn list_tasks(&self) {
        let tasks = self.tasks.borrow();
        for (i, task) in tasks.iter().enumerate() {
            let status = if task.completed { "✓" } else { "○" };
            println!("{}. {} {}", i + 1, status, task.description);
        }
    }
}

fn main() {
    let todo = TodoList::new();
    
    let task1 = todo.add_task(String::from("学习 Rust 所有权"));
    let task2 = todo.add_task(String::from("练习生命周期"));
    
    todo.list_tasks();
    
    // 可以在其他地方持有任务的引用
    println!("任务 1: {:?}", task1);
    println!("任务 2: {:?}", task2);
    
    // task1 和 task2 离开作用域后，todo 仍然持有它们的引用
    drop(task1);
    drop(task2);
    
    println!("\n删除外部引用后：");
    todo.list_tasks();
}

关键概念理解

所有权决策树 

1
2
3
4
5

函数需要修改数据吗？
├─ 是 → 使用 &mut T
└─ 否 → 函数需要消耗值吗？
    ├─ 是 → 获取所有权 T
    └─ 否 → 使用 &T 或 &str

生命周期标注规则

  1. 省略规则：每个引用参数获得独立的生命周期参数
  2. 返回值规则：如果只有一个输入生命周期，它被赋给所有输出生命周期
  3. 方法规则：如果 &self 或 &mut self 存在，self 的生命周期赋给所有输出生命周期

Rc vs RefCell 对比

特性 Rc RefCell
借用检查 编译时 运行时
可变性 不可变 可内部可变
性能开销 引用计数 运行时检查
典型用途 多所有权 内部可变性
组合使用 Rc<RefCell> 多所有者 + 可变

今日感悟

今天的所有权应用模式学习让我对 Rust 的设计哲学有了更深的理解：

  1. 所有权是工具，不是束缚：最初觉得所有权规则繁琐，但现在明白它是为了在编译时消除一大类 bug。每次编译器报错，都是在帮我避免运行时的内存问题。

  2. 生命周期的本质：生命周期标注不是魔法，而是帮助编译器理解引用关系的工具。一旦理解了"返回的引用不能比输入活得更久"这个基本原则，标注就变得直观了。

  3. Rc 和 RefCell 的权衡：这两个类型提供了灵活性，但也放弃了编译时的部分保证。最佳实践是优先使用所有权和借用，只在确实需要时才用智能指针。

  4. 实践出真知：看再多理论也不如写代码。今天的待办事项示例让我真正理解了如何将所有权、Rc、RefCell 组合使用。

遇到的挑战主要是生命周期的思维转换。习惯了其他语言的垃圾回收后，需要重新建立对数据生命周期的敏感度。但一旦掌握，写出的代码会更加健壮和高效。

明天的结构体学习应该会更有趣，可以开始设计更复杂的数据结构了！
