为什么测试很重要？

在开始写测试代码之前，先理解为什么测试如此重要：

  1. 防止回归 - 确保新代码不会破坏已有功能
  2. 文档作用 - 测试用例展示了代码应该如何被使用
  3. 设计反馈 - 难以测试的代码通常设计有问题
  4. 重构信心 - 有了测试，重构时更有底气
  5. 捕获边界情况 - 强制思考各种输入场景

Rust 内置了强大的测试支持，让编写和运行测试变得非常简单。

pub fn divide(a: i32, b: i32) -> Result<i32, String> {
    if b == 0 {
        Err("除数不能为零".to_string())
    } else {
        Ok(a / b)
    }
}

pub fn find_first_even(numbers: &[i32]) -> Option<i32> {
    numbers.iter().find(|&&x| x % 2 == 0).copied()
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_divide_success() {
        let result = divide(10, 2);
        assert_eq!(result, Ok(5));
    }

    #[test]
    fn test_divide_by_zero() {
        let result = divide(10, 0);
        assert!(result.is_err());
        assert_eq!(result.unwrap_err(), "除数不能为零");
    }

    #[test]
    fn test_find_first_even_found() {
        let numbers = vec![1, 3, 5, 8, 9];
        let result = find_first_even(&numbers);
        assert_eq!(result, Some(8));
    }

    #[test]
    fn test_find_first_even_not_found() {
        let numbers = vec![1, 3, 5, 7, 9];
        let result = find_first_even(&numbers);
        assert_eq!(result, None);
    }

    // 使用 ? 运算符简化 Result 测试
    #[test]
    fn test_divide_with_question_mark() -> Result<(), String> {
        let result = divide(15, 3)?;
        assert_eq!(result, 5);
        Ok(())
    }
}

#[test]
fn test_equality_assertions() {
    let a = 5;
    let b = 5;
    let c = 10;
    
    assert_eq!(a, b);        // 断言相等
    assert_ne!(a, c);        // 断言不相等
    
    // 自定义错误消息
    assert_eq!(a, b, "a 和 b 应该相等，但 a={} b={}", a, b);
    
    // 可以比较复杂类型
    let vec1 = vec![1, 2, 3];
    let vec2 = vec![1, 2, 3];
    assert_eq!(vec1, vec2);
    
    let str1 = "hello";
    let str2 = "hello";
    assert_eq!(str1, str2);
}

    // 这个测试不应该 panic

/// 一个简单的计算器模块
pub mod calculator {
    pub fn add(a: i32, b: i32) -> i32 {
        a + b
    }

    pub fn subtract(a: i32, b: i32) -> i32 {
        a - b
    }

    pub fn multiply(a: i32, b: i32) -> i32 {
        a * b
    }

    pub fn divide(a: i32, b: i32) -> Result<i32, String> {
        if b == 0 {
            Err("除数不能为零".to_string())
        } else {
            Ok(a / b)
        }
    }

    pub fn power(base: i32, exp: u32) -> i32 {
        base.pow(exp)
    }

    pub fn factorial(n: u32) -> Result<u32, String> {
        if n > 20 {
            Err("n 太大，会导致溢出".to_string())
        } else {
            Ok((1..=n).product())
        }
    }
}

/// 字符串工具模块
pub mod string_utils {
    pub fn reverse(s: &str) -> String {
        s.chars().rev().collect()
    }

    pub fn is_palindrome(s: &str) -> bool {
        let cleaned: String = s
            .chars()
            .filter(|c| c.is_alphanumeric())
            .collect::<String>()
            .to_lowercase();
        
        cleaned == cleaned.chars().rev().collect::<String>()
    }

    pub fn count_words(s: &str) -> usize {
        s.split_whitespace().count()
    }
}

        #[test]
        fn test_subtract() {
            assert_eq!(calculator::subtract(5, 3), 2);
            assert_eq!(calculator::subtract(3, 5), -2);

            assert_eq!(calculator::multiply(3, 4), 12);
            assert_eq!(calculator::multiply(-2, 3), -6);
            assert_eq!(calculator::multiply(0, 100), 0);
        }

        #[test]
        fn test_divide_success() {
            assert_eq!(calculator::divide(10, 2), Ok(5));
            assert_eq!(calculator::divide(7, 2), Ok(3)); // 整数除法
        }

        #[test]
        fn test_divide_by_zero() {
            let result = calculator::divide(10, 0);
            assert!(result.is_err());
            assert_eq!(result.unwrap_err(), "除数不能为零");
        }

        #[test]
        fn test_power() {
            assert_eq!(calculator::power(2, 3), 8);
            assert_eq!(calculator::power(5, 0), 1);
            assert_eq!(calculator::power(2, 10), 1024);
        }

        #[test]
        fn test_factorial() {
            assert_eq!(calculator::factorial(0), Ok(1));
            assert_eq!(calculator::factorial(1), Ok(1));
            assert_eq!(calculator::factorial(5), Ok(120));
            assert_eq!(calculator::factorial(10), Ok(3628800));
        }

        #[test]
        fn test_factorial_overflow() {
            let result = calculator::factorial(21);
            assert!(result.is_err());
        }
    }

    mod string_utils_tests {
        use super::*;

        #[test]
        fn test_reverse() {
            assert_eq!(string_utils::reverse("hello"), "olleh");
            assert_eq!(string_utils::reverse(""), "");

        }

        #[test]
        fn test_is_palindrome() {
            assert!(string_utils::is_palindrome("racecar"));
            assert!(string_utils::is_palindrome("A man a plan a canal Panama"));
            assert!(!string_utils::is_palindrome("hello"));

fn test_calculator_chain() {
    // 测试多个计算器函数的链式调用
    let a = 10;
    let b = 5;

fn test_calculator_power_and_multiply() {
    let base = 2;
    let exp = 3;
    
    let powered = calculator::power(base, exp);
    let result = calculator::multiply(powered, powered);
    
    assert_eq!(result, 64); // 2^3 * 2^3 = 8 * 8 = 64
}

今日感悟

今天正式开始学习 Rust 的测试系统，有几个深刻的体会：

测试是 Rust 的一等公民：与其他语言需要额外配置测试框架不同，Rust 内置了完整的测试支持。#[test] 属性、断言宏、cargo test 命令，一切都开箱即用。这种设计让写测试变得非常简单，几乎没有门槛。

单元测试可以访问私有函数：这一点很赞！在很多语言中，测试私有函数需要特殊技巧或者根本做不到。但 Rust 允许测试模块访问同一模块中的所有内容，这意味着我们可以测试内部实现细节，而不仅仅是公共 API。这对于确保复杂内部逻辑的正确性很有帮助。

断言宏的智能输出：assert_eq! 和 assert_ne! 在失败时会显示两个值的差异，这对于调试非常有帮助。不需要手动拼接错误消息就能看到问题所在。

测试隔离的重要性：每个测试都应该是独立的，不依赖其他测试的状态。这意味着不能依赖全局变量或者测试执行顺序。这个约束虽然严格，但能写出更健壮的测试。

集成测试与单元测试的分工：单元测试关注内部逻辑，集成测试关注公共 API 的组合使用。两者互补，共同保证代码质量。

明天将继续深入学习测试进阶内容，包括参数化测试和覆盖率分析。测试是保证代码质量的重要手段，值得投入时间深入学习。
