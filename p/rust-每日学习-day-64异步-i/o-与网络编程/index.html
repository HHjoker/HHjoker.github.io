回顾：Tokio 运行时

昨天我们深入学习了 Tokio 运行时架构：

  • multi_thread：多线程运行时，适用于 CPU 密集和并发任务
  • current_thread：单线程运行时，适合轻量级场景
  • spawn：轻量级任务调度，非阻塞
  • spawn_blocking：用于阻塞操作，自动转移到专用线程池

今天我们继续深入，学习异步 I/O 和网络编程。

一、异步 I/O 基础

为什么需要异步 I/O？

传统的同步 I/O 在等待期间会阻塞线程：

1
2

// 同步方式：等待期间线程被阻塞
let data = file.read();  // 等待读取完成，线程无法做其他事

异步 I/O 允许在等待期间执行其他任务：

1
2

// 异步方式：等待期间可以做其他事
let data = async_file.read().await;  // 让出线程，其他任务可以运行

Tokio 的 I/O 模块

Tokio 提供了 tokio::io 模块，支持异步文件和网络 I/O 操作。

use tokio::io::{AsyncReadExt, AsyncWriteExt};
use tokio::fs::File;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    // 创建测试文件
    let mut file = File::create("hello.txt").await?;
    file.write_all(b"Hello, Rust async I/O!").await?;
    
    // 读取文件内容
    let mut file = File::open("hello.txt").await?;
    let mut contents = String::new();
    file.read_to_string(&mut contents).await?;
    
    println!("文件内容: {}", contents);
    
    // 清理
    tokio::fs::remove_file("hello.txt").await?;
    
    Ok(())
}

关键点解释：

  • tokio::fs::File：异步文件操作，与标准库 std::fs::File 类似
  • AsyncWriteExt：提供 write_all 等异步写入方法
  • AsyncReadExt：提供 read_to_string 等异步读取方法

use tokio::io::{AsyncBufReadExt, BufReader};
use tokio::fs::File;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    // 创建测试文件
    let file = File::create("lines.txt").await?;
    let mut writer = BufWriter::new(file);
    writer.write_all(b"Line 1\nLine 2\nLine 3\n").await?;
    writer.flush().await?;
    
    // 逐行读取
    let file = File::open("lines.txt").await?;
    let reader = BufReader::new(file);
    let mut lines = reader.lines();
    
    while let Some(line) = lines.next_line().await? {
        println!("读取到一行: {}", line);
    }
    
    // 清理
    tokio::fs::remove_file("lines.txt").await?;
    
    Ok(())
}

use tokio::io::{AsyncReadExt, AsyncWriteExt};
use tokio::net::TcpListener;
use tokio::io::BufReader;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    // 创建 TCP 监听器
    let listener = TcpListener::bind("127.0.0.1:8080").await?;
    println!("服务器监听在 127.0.0.1:8080");
    
    loop {
        // 接受新连接
        let (mut socket, addr) = listener.accept().await?;
        println!("收到来自 {} 的连接", addr);
        
        // 为每个连接创建一个任务
        tokio::spawn(async move {
            // 读取数据
            let mut buf = [0u8; 1024];
            let n = socket.read(&mut buf).await.unwrap_or(0);
            
            if n > 0 {
                // 发送响应
                let response = format!("收到 {} 字节数据", n);
                socket.write_all(response.as_bytes()).await.unwrap_or(0);
            }
            println!("与 {} 的连接关闭", addr);
        });
    }
}

use tokio::io::{AsyncReadExt, AsyncWriteExt};
use tokio::net::TcpListener;
use bytes::{BytesMut, BufMut};

// 使用固定长度头部（4字节长度 + 数据）
#[derive(Clone)]
struct FramedConfig {
    max_frame_size: usize,
}

impl FramedConfig {
    fn new() -> Self {
        Self {
            max_frame_size: 1024 * 1024, // 最大 1MB
        }
    }
    
    // 编码消息：写入长度前缀 + 数据
    fn encode(&self, msg: &str, buf: &mut BytesMut) -> Result<(), &'static str> {
        let data = msg.as_bytes();
        let len = data.len() as u32;
        
        // 检查大小限制
        if len > self.max_frame_size as u32 {
            return Err("消息太大");
        }
        
        // 写入长度（4字节大端序）
        buf.put_u32(len);
        // 写入数据
        buf.put_slice(data);
        Ok(())
    }
}

// 服务器处理函数
async fn handle_client(
    mut socket: tokio::net::TcpStream,
    config: FramedConfig
) -> Result<(), Box<dyn std::error::Error>> {
    let mut buf = BytesMut::new();
    loop {
        // 读取数据
        let n = socket.read_buf(&mut buf).await?;
        if n == 0 {
            break; // 连接关闭
        }
        
        // 处理消息（简化版，假设完整消息）
        while buf.len() >= 4 {
            // 读取长度
            let mut size_buf = [0u8; 4];
            size_buf.copy_from_slice(&buf[..4]);
            let len = u32::from_be_bytes(size_buf) as usize;
            
            if buf.len() < 4 + len {
                break; // 消息不完整
            }
            
            // 提取消息
            buf.advance(4);
            let msg = String::from_utf8_lossy(&buf[..len]).to_string();
            buf.advance(len);
            
            println!("收到消息: {}", msg);
            
            // 发送响应
            let response = format!("服务端收到: {}", msg);
            let mut response_buf = BytesMut::new();
            config.encode(&response, &mut response_buf)?;
            socket.write_all(&response_buf).await?;
        }
    }
    Ok(())
}

use tokio::net::UdpSocket;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    // 创建 UDP 监听
    let socket = UdpSocket::bind("127.0.0.1:8080").await?;
    println!("UDP 服务器监听在 127.0.0.1:8080");
    
    let mut buf = [0u8; 1024];
    loop {
        // 接收数据
        let (n, addr) = socket.recv_from(&mut buf).await?;
        let data = String::from_utf8_lossy(&buf[..n]);
        println!("收到来自 {} 的数据: {}", addr, data);
        
        // 发送响应
        let response = format!("收到 {} 字节", n);
        socket.send_to(response.as_bytes(), &addr).await?;
    }
}

// UDP 客户端
async fn udp_client() -> Result<(), Box<dyn std::error::Error>> {
    let socket = UdpSocket::bind("127.0.0.1:0").await?;
    socket.connect("127.0.0.1:8080").await?;
    
    // 发送数据
    socket.send(b"Hello, UDP Server!").await?;
    println!("数据已发送");
    
    // 接收响应
    let mut buf = [0u8; 1024];
    let n = socket.recv(&mut buf).await?;
    let response = String::from_utf8_lossy(&buf[..n]);
    println!("收到响应: {}", response);
    
    Ok(())
}

use tokio::time::{timeout, Duration};
use tokio::net::TcpStream;

async fn connect_with_timeout() -> Result<(), Box<dyn std::error::Error>> {
    // 最多等待 5 秒
    let result = timeout(
        Duration::from_secs(5),
        TcpStream::connect("127.0.0.1:8080")
    ).await;
    
    match result {
        Ok(Ok(stream)) => {
            println!("连接成功!");
            Ok(())
        }
        Ok(Err(e)) => {
            println!("连接失败: {}", e);
            Err(Box::new(e))
        }
        Err(_) => {
            println!("连接超时!");
            Err("连接超时".into())
        }
    }
}

use tokio::time::{sleep, Duration};

async fn retry_with_backoff<F, Fut, T>(
    mut operation: F,
    max_retries: u32,
    initial_delay: Duration,
) -> Result<T, &'static str>
where
    F: FnMut() -> Fut,
    Fut: std::future::Future<Output = Result<T, &'static str>>,
{
    let mut delay = initial_delay;
    
    for attempt in 0..max_retries {
        match operation().await {
            Ok(result) => return Ok(result),
            Err(e) if attempt < max_retries - 1 => {
                println!("尝试 {} 失败: {}，{}ms 后重试...", attempt + 1, e, delay.as_millis());
                sleep(delay).await;
                delay *= 2; // 指数退避
            }
            Err(e) => return Err(e),
        }
    }
    
    Err("达到最大重试次数")
}

// 使用示例
async fn example() {
    let result = retry_with_backoff(
        || async {
            // 模拟可能失败的操作
            if rand::random::<f32>() < 0.7 {
                Err("网络错误")
            } else {
                Ok("成功")
            }
        },
        3,
        Duration::from_millis(100),
    ).await;
    
    println!("结果: {:?}", result);
}

use tokio::io::{AsyncReadExt, AsyncWriteExt};
use tokio::net::{TcpListener, UdpSocket};
use std::sync::Arc;

struct EchoConfig {
    tcp_port: u16,
    udp_port: u16,
}

async fn tcp_echo_server(port: u16) -> Result<(), Box<dyn std::error::Error>> {
    let listener = TcpListener::bind(format!("127.0.0.1:{}", port)).await?;
    println!("TCP Echo 服务器监听在端口 {}", port);
    
    loop {
        let (mut socket, addr) = listener.accept().await?;
        tokio::spawn(async move {
            println!("TCP 连接: {}", addr);
            
            let mut buf = vec![0u8; 1024];
            loop {
                let n = match socket.read(&mut buf).await {
                    Ok(n) if n == 0 => break,
                    Ok(n) => n,
                    Err(e) => {
                        eprintln!("读取错误: {}", e);
                        break;
                    }
                };
                
                if let Err(e) = socket.write_all(&buf[..n]).await {
                    eprintln!("写入错误: {}", e);
                    break;
                }
            }
            
            println!("TCP 连接关闭: {}", addr);
        });
    }
}

async fn udp_echo_server(port: u16) -> Result<(), Box<dyn std::error::Error>> {
    let socket = UdpSocket::bind(format!("127.0.0.1:{}", port)).await?;
    println!("UDP Echo 服务器监听在端口 {}", port);
    
    let mut buf = [0u8; 1024];
    loop {
        let (n, addr) = socket.recv_from(&mut buf).await?;
        println!("UDP 数据来自 {}: {} 字节", addr, n);
        socket.send_to(&buf[..n], addr).await?;
    }
}

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let config = Arc::new(EchoConfig {
        tcp_port: 8080,
        udp_port: 8081,
    });
    
    // 同时运行 TCP 和 UDP 服务器
    let tcp_config = config.clone();
    let udp_config = config.clone();
    
    let tcp_handle = tokio::spawn(async move {
        tcp_echo_server(tcp_config.tcp_port).await
    });
    
    let udp_handle = tokio::spawn(async move {
        udp_echo_server(udp_config.udp_port).await
    });
    
    // 等待任一服务器结束
    tokio::select! {
        result = tcp_handle => {
            if let Err(e) = result {
                eprintln!("TCP 服务器错误: {}", e);
            }
        }
        result = udp_handle => {
            if let Err(e) = result {
                eprintln!("UDP 服务器错误: {}", e);
            }
        }
    }
    
    Ok(())
}

关键概念理解

1. 异步 I/O 的优势 

1
2
3
4
5
6

同步 I/O:                      异步 I/O:
[请求1]---------> [等待]       [请求1]---(await)--->
         \                       \ 
          [请求2]---(等待)        [请求2]---(await)--->
          
总时间: 请求1 + 等待1 + 请求2  总时间: max(请求1, 请求2)

2. Tokio I/O 模型

  • 非阻塞操作：异步 I/O 操作不会阻塞线程
  • 任务调度：I/O 等待期间，线程可以执行其他任务
  • 系统集成： Tokio 使用操作系统提供的 epoll/kqueue/IOCP

3. 网络编程要点

  • TCP 是流协议，需要自己处理消息边界
  • UDP 是数据报协议，有消息边界
  • 使用 spawn 处理并发连接
  • 添加超时和重试提高可靠性

今日感悟

今天学习了 Tokio 的异步 I/O 和网络编程。网络编程是 Rust 异步生态中最成熟的应用场景之一。通过 TCP 服务器和 UDP 客户端的实现，我深刻理解了异步网络模型的优势。

关键收获：

  1. I/O 操作的异步化让线程在等待时可以做其他工作
  2. TCP 需要处理消息边界，UDP 本身有消息边界
  3. spawn 用于处理并发连接，每个连接一个轻量级任务
  4. 超时和重试是网络编程中不可或缺的模式

网络编程是异步 Rust 的核心应用场景，熟练掌握这些模式对后续学习非常重要。
