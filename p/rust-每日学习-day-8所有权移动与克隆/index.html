学习内容

1. 所有权移动（Move）

在 Rust 中，当一个值被赋值给另一个变量或传递给函数时，所有权会发生移动，而不是复制。这是 Rust 内存安全的核心机制。

1
2
3
4
5
6
7

fn main() {
    let s1 = String::from("hello");
    let s2 = s1;  // 所有权从 s1 移动到 s2
    
    // println!("{}", s1);  // 编译错误！s1 已不再有效
    println!("{}", s2);  // 正确，s2 拥有这个字符串
}

为什么这样设计？

Rust 不允许两个变量同时拥有同一块堆内存的所有权。如果允许，当两个变量离开作用域时，它们都会尝试释放同一块内存，导致**双重释放（double free）**错误。

2. 深层克隆（Clone）

如果你确实需要复制数据而不是移动所有权，可以使用 clone() 方法：

1
2
3
4
5
6

fn main() {
    let s1 = String::from("hello");
    let s2 = s1.clone();  // 深层克隆，创建新的堆内存
    
    println!("s1 = {}, s2 = {}", s1, s2);  // 两个都可以使用
}

移动 vs 克隆的对比：

操作 内存行为 性能 使用场景
移动（Move） 所有权转移，不复制数据 零成本 默认行为，高效
克隆（Clone） 创建数据的完整副本 有开销 需要两份独立数据时

3. Copy trait

对于存储在栈上的简单类型，Rust 实现了 Copy trait，这些类型在赋值时会自动复制：

1
2
3
4
5
6

fn main() {
    let x = 5;
    let y = x;  // x 被复制，而不是移动
    
    println!("x = {}, y = {}", x, y);  // 两个都可以使用
}

实现 Copy 的类型：

  • 所有整数类型（i32, u64, 等）
  • 布尔类型 bool
  • 浮点类型（f32, f64）
  • 字符类型 char
  • 只包含 Copy 类型的元组

fn main() {
    let s = String::from("hello");

}

5. 借用（Borrowing）预览

如果不想移动所有权，可以使用引用（下一课详细讲解）：

fn main() {
    let s1 = String::from("hello");
    let len = calculate_length(&s1);  // 借用，不移动所有权
    
    println!("'{}' 的长度是 {}", s1, len);  // s1 仍然可用
}

fn calculate_length(s: &String) -> usize {
    s.len()
}

关键概念理解

🎯 核心要点

  1. 移动是默认的：复杂类型（如 String）赋值时所有权移动
  2. Copy 是特例：简单类型自动复制，因为它们只在栈上
  3. Clone 是显式的：需要深层复制时调用 .clone()
  4. 移动后不可用：所有权移动后，原变量不能再使用

🧠 内存模型

移动前：
s1 -> [堆内存 "hello"]

移动后 (let s2 = s1)：
s1 -> (无效)
s2 -> [堆内存 "hello"]

克隆后 (let s2 = s1.clone())：
s1 -> [堆内存 "hello"]
s2 -> [堆内存 "hello"] (新的副本)

练习 4：Copy 类型判断

判断以下类型是否实现 Copy trait：

  1. i32
  2. String
  3. bool
  4. Vec<i32>
  5. (i32, i32)
  6. &str
点击查看答案
  1. i32 ✓ 是
  2. String ✗ 否（堆分配）
  3. bool ✓ 是
  4. Vec<i32> ✗ 否（堆分配）
  5. (i32, i32) ✓ 是（所有元素都是 Copy）
  6. &str ✓ 是（引用是 Copy 的）

今日感悟

所有权移动是 Rust 最反直觉但也最强大的特性之一。

刚开始学习 Rust 时，很多人（包括我）会对"为什么赋值后原变量就不能用了"感到困惑。这是因为我们习惯了其他语言的复制语义。

但一旦理解了这背后的原因——防止双重释放、保证内存安全——就会明白这是非常优雅的设计。Rust 在编译期就解决了其他语言需要垃圾回收器或手动管理才能解决的问题。

关键领悟：

  • 移动是零成本的，克隆才有开销
  • 默认移动，需要复制时显式 clone
  • 简单类型自动复制（Copy trait）
  • 这是 Rust 内存安全的基石

明天学习引用和借用，那是解决"想使用但不想移动"问题的终极方案！

继续前进，Rust 的世界越来越清晰了！ 🦀
