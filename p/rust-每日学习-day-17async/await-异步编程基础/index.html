同步 vs 异步：

// 同步代码：按顺序执行，每一步等待上一步完成
fn sync_example() {
    println!("步骤 1: 开始");
    std::thread::sleep(std::time::Duration::from_secs(2));  // 阻塞 2 秒
    println!("步骤 2: 完成");
}

// 异步代码：启动任务后不等待，可以继续做其他事
async fn async_example() {
    println!("步骤 1: 开始");
    tokio::time::sleep(tokio::time::Duration::from_secs(2)).await;  // 非阻塞
    println!("步骤 2: 完成");
}

为什么需要异步？

场景 同步方式 异步方式
网络请求 阻塞等待响应 等待时可处理其他请求
文件 I/O 阻塞等待读取 等待时可处理其他任务
数据库查询 阻塞等待结果 等待时可处理其他查询
高并发服务器 需要大量线程 少量线程处理大量连接

核心优势：

  • 🚀 更高的并发性能
  • 💾 更少的内存占用（不需要为每个连接创建线程）
  • ⚡ 更好的资源利用率

use std::future::Future;
use std::pin::Pin;
use std::task::{Context, Poll};

/// Future trait 的定义（简化版）
pub trait Future {
    type Output;
    
    /// 轮询 Future 是否完成
    fn poll(self: Pin<&mut Self>, cx: &mut Context<'_>) -> Poll<Self::Output>;
}

/// Poll 枚举表示 Future 的状态
pub enum Poll<T> {
    Ready(T),      // 已完成，返回结果
    Pending,       // 未完成，需要继续等待
}

use std::future::Future;
use std::pin::Pin;
use std::task::{Context, Poll};
use std::time::{Duration, Instant};

/// 一个简单的延迟 Future
struct Delay {
    when: Instant,
}

impl Future for Delay {
    type Output = ();
    
    fn poll(self: Pin<&mut Self>, cx: &mut Context<'_>) -> Poll<Self::Output> {
        // 检查时间是否已到
        if Instant::now() >= self.when {
            println!("⏰ 时间到！");
            Poll::Ready(())  // 完成
        } else {
            // 时间未到，注册 waker 并返回 Pending
            cx.waker().wake_by_ref();
            Poll::Pending
        }
    }
}

/// 创建 Delay Future 的辅助函数
fn delay(duration: Duration) -> Delay {
    Delay {
        when: Instant::now() + duration,
    }
}

#[tokio::main]
async fn main() {
    println!("🚀 开始等待...");
    delay(Duration::from_secs(2)).await;
    println!("✅ 等待完成！");
}

use tokio::time::{sleep, Duration};

/// 模拟网页抓取
async fn fetch_page(url: &str) -> Result<String, String> {
    println!("🕷️  抓取：{}", url);
    
    // 模拟网络延迟
    sleep(Duration::from_millis(100)).await;
    
    // 模拟可能的失败
    if url.contains("error") {
        return Err(format!("无法访问：{}", url));
    }
    
    Ok(format!("<html>{}</html>", url))
}

/// 处理单个 URL
async fn process_url(url: &str) {
    match fetch_page(url).await {
        Ok(content) => {
            println!("✅ 成功：{} ({} 字节)", url, content.len());
        }
        Err(e) => {
            println!("❌ 失败：{}", e);
        }
    }
}

#[tokio::main]
async fn main() {
    let urls = vec![
        "https://example.com",
        "https://rust-lang.org",
        "https://tokio.rs",
        "https://error-page.com",  // 模拟失败
        "https://github.com",
    ];
    
    println!("🚀 开始爬取 {} 个页面...\n", urls.len());
    
    // 并发抓取所有页面
    let handles: Vec<_> = urls
        .iter()
        .map(|url| tokio::spawn(process_url(url)))
        .collect();
    
    // 等待所有任务完成
    for handle in handles {
        handle.await.unwrap();
    }
    
    println!("\n🎉 爬取完成！");
}

// 并发（Concurrency）：多个任务交替执行
// 并行（Parallelism）：多个任务同时执行

async fn concurrent_example() {
    // 两个任务并发执行（可能在不同线程，也可能在同一线程交替）
    let (a, b) = tokio::join!(
        task_a(),  // 不阻塞，立即返回 Future
        task_b()   // 不阻塞，立即返回 Future
    );
}

// 使用 spawn 可以在多线程上并行执行
async fn parallel_example() {
    let handle_a = tokio::spawn(task_a());  // 可能在另一个线程
    let handle_b = tokio::spawn(task_b());  // 可能在另一个线程
    
    let a = handle_a.await.unwrap();
    let b = handle_b.await.unwrap();
}

// 同步：每个连接需要一个线程
fn sync_server() {
    for stream in incoming_connections() {
        std::thread::spawn(|| {
            handle_connection(stream);  // 阻塞
        });
    }
}

// 异步：一个线程处理多个连接
async fn async_server() {
    while let Some(stream) = incoming_connections().await {
        tokio::spawn(async move {
            handle_connection(stream).await;  // 非阻塞
        });
    }
}

今日感悟

🌟 异步编程：新的思维方式

今天正式踏入了 Rust 异步编程的大门，这确实是一个思维方式的转变。

最大的收获：

  1. Future 的本质：Future 不是"将来的值"，而是"计算过程的表示"
  2. 惰性求值：Future 不执行就不会有任何效果，必须 .await
  3. 非阻塞 I/O：这是异步编程的核心优势，一个线程可以处理成千上万的连接
  4. Tokio 的强大：提供了完整的异步生态系统

遇到的困难：

  • 理解 .await 到底做了什么（挂起、让出执行权）
  • Send trait 的限制（为什么有些类型不能跨线程）
  • 调试异步代码比同步代码更复杂

有趣的发现：

  • async fn 返回的不是值，而是 Future
  • tokio::join! 和 tokio::spawn 的区别（前者并发，后者并行）
  • tokio::select! 可以优雅地处理超时和取消

给初学者的建议：

  1. 先理解同步代码，再学异步
  2. 多用 tokio::spawn 和 join! 练习
  3. 不要过早优化，先让代码跑起来
  4. 阅读 Tokio 的官方教程，非常详细

下一步：

  • 学习 Stream（异步版本的 Iterator）
  • 掌握异步通道（channel）
  • 实践：用异步构建一个真实的网络应用

异步编程是 Rust 进阶的必经之路，虽然学习曲线陡峭，但掌握后的收益是巨大的。继续加油！🦀
