今日内容

经过前几天的学习，我们已经掌握了 async/await 的核心概念：Future 的 poll 机制、Tokio 运行时、join!/select! 等并发原语。今天我们把这些知识串联起来，使用 Rust 最流行的异步 Web 框架 axum 构建一个真实的 HTTP 服务器。

1. 为什么选择 axum？

axum 是目前 Rust 生态中最受好评的 Web 框架之一，原因如下：

  • 基于 Tower 和 Tower-STACK：天然支持中间件、服务发现等生态
  • 类型安全的路由：用泛型保证路由参数的正确性
  • 与 Tokio 完全集成：直接使用 async/await，无需额外学习
  • 性能出色：对标 Go 的 net/http 和 Node.js 的 Express
  • 活跃社区：主流的 API 框架如 tambo、launchbadge 均基于 axum

use axum::{routing::get, Router};
use std::net::SocketAddr;

#[tokio::main]
async fn main() {
    // 定义路由：GET / → hello_handler
    let app = Router::new().route("/", get(hello_handler));

    let addr = SocketAddr::from(([127, 0, 0, 1], 3000));
    let listener = tokio::net::TcpListener::bind(addr).await.unwrap();
    println!("服务器运行在 http://{addr}");

    axum::serve(listener, app).await.unwrap();
}

// 处理器函数：返回文本
async fn hello_handler() -> &'static str {
    "Hello, Rust async world!"
}

运行后访问 http://127.0.0.1:3000/，浏览器会显示 “Hello, Rust async world!"。

代码解读：

  1. Router::new() 创建一个空的路由表
  2. .route("/", get(hello_handler)) 注册 GET / 路由，绑定处理器
  3. axum::serve() 接收 TCP listener 和路由，将请求分发给对应处理器
  4. 处理器是 async fn，由 Tokio 运行时自动调度

use axum::{routing::get, Router, extract::{Path, Query, Json}};
use serde::Deserialize;

#[derive(Debug, Deserialize)]
struct Pagination {
    page: Option<usize>,
    per_page: Option<usize>,
}

async fn get_user(Path(user_id): Path<u32>, Query(params): Query<Pagination>) -> String {
    let page = params.page.unwrap_or(1);
    let per_page = params.per_page.unwrap_or(10);
    format!("获取用户 ID: {user_id}, 第 {page} 页, 每页 {per_page} 条", )
}

#[tokio::main]
async fn main() {
    let app = Router::new()
        .route("/users/:id", get(get_user));

    let listener = tokio::net::TcpListener::bind("127.0.0.1:3000").await.unwrap();
    axum::serve(listener, app).await.unwrap();
}

访问 http://127.0.0.1:3000/users/42?page=2&per_page=20，输出：

1

获取用户 ID: 42, 第 2 页, 每页 20 条

代码解读：

  • Path<u32> 从 URL 路径提取 user_id，自动转换为 u32 类型
  • Query<Pagination> 从查询字符串解析分页参数
  • extract 是 axum 的核心概念：处理器函数通过参数"提取"请求中的各个部分

use axum::{
    routing::{get, post},
    Router,
    extract::{Path, Json},
    response::IntoResponse,
};
use serde::{Deserialize, Serialize};

#[derive(Debug, Deserialize)]
struct CreateUser {
    name: String,
    email: String,
}

#[derive(Debug, Serialize)]
struct User {
    id: u32,
    name: String,
    email: String,
}

#[derive(Debug, Serialize)]
struct ApiResponse<T> {
    success: bool,
    data: Option<T>,
    message: &'static str,
}

// GET /users/:id
async fn get_user(Path(user_id): Path<u32>) -> impl IntoResponse {
    // 模拟数据库查找
    if user_id == 1 {
        Json(ApiResponse {
            success: true,
            data: Some(User {
                id: 1,
                name: "Alice".to_string(),
                email: "alice@example.com".to_string(),
            }),
            message: "success",
        })
    } else {
        (axum::http::StatusCode::NOT_FOUND, Json(ApiResponse {
            success: false,
            data: None,
            message: "User not found",
        }))
    }
}

// POST /users
async fn create_user(Json(payload): Json<CreateUser>) -> impl IntoResponse {
    let new_user = User {
        id: 100, // 模拟自增 ID
        name: payload.name,
        email: payload.email,
    };

    (axum::http::StatusCode::CREATED, Json(ApiResponse {
        success: true,
        data: Some(new_user),
        message: "User created",
    }))
}

#[tokio::main]
async fn main() {
    let app = Router::new()
        .route("/users/:id", get(get_user))
        .route("/users", post(create_user));

    let listener = tokio::net::TcpListener::bind("127.0.0.1:3000").await.unwrap();
    axum::serve(listener, app).await.unwrap();
}

use axum::{
    routing::get,
    Router,
    middleware,
};
use tower_http::trace::TraceLayer;
use tower_http::cors::{CorsLayer, Any};

async fn handler() -> &'static str {
    "响应内容"
}

#[tokio::main]
async fn main() {
    let app = Router::new()
        .route("/", get(handler))
        // 日志追踪中间件：打印每个请求的 method、path、status、耗时
        .layer(TraceLayer::new_for_http())
        // CORS 中间件：允许所有来源、所有方法、所有头部
        .layer(CorsLayer::new()
            .allow_origin(Any)
            .allow_methods(Any)
            .allow_headers(Any)
        );

    let listener = tokio::net::TcpListener::bind("127.0.0.1:3000").await.unwrap();
    axum::serve(listener, app).await.unwrap();
}

use axum::{
    routing::get,
    Router,
    extract::Path,
    response::{IntoResponse, Response},
    Json,
};
use axum::http::StatusCode;
use serde_json::json;

#[derive(Debug)]
enum AppError {
    NotFound(String),
    BadRequest(String),
    InternalError(String),
}

impl IntoResponse for AppError {
    fn into_response(self) -> Response {
        let (status, message) = match self {
            AppError::NotFound(msg) => (StatusCode::NOT_FOUND, msg),
            AppError::BadRequest(msg) => (StatusCode::BAD_REQUEST, msg),
            AppError::InternalError(msg) => (StatusCode::INTERNAL_SERVER_ERROR, msg),
        };

        (status, Json(json!({
            "success": false,
            "error": message
        }))).into_response()
    }
}

async fn find_item(Path(id): Path<u32>) -> Result<String, AppError> {
    if id == 0 {
        Err(AppError::BadRequest("ID 不能为 0".to_string()))
    } else if id > 1000 {
        Err(AppError::NotFound(format!("商品 ID {id} 不存在")))
    } else {
        Ok(format!("找到商品: #{}", id))
    }
}

#[tokio::main]
async fn main() {
    let app = Router::new().route("/items/:id", get(find_item));
    let listener = tokio::net::TcpListener::bind("127.0.0.1:3000").await.unwrap();
    axum::serve(listener, app).await.unwrap();
}

use axum::{
    routing::get,
    Router,
    extract::State,
    response::Json,
};
use std::sync::Arc;
use tokio::sync::RwLock;
use serde::Serialize;

// 共享的可变状态
#[derive(Default)]
struct AppState {
    request_count: RwLock<u64>,
    items: RwLock<Vec<String>>,
}

#[derive(Serialize)]
struct StatsResponse {
    request_count: u64,
    items_count: usize,
}

async fn stats(State(state): State<Arc<AppState>>) -> Json<StatsResponse> {
    let count = *state.request_count.read().await;
    let items_count = state.items.read().await.len();
    Json(StatsResponse {
        request_count: count,
        items_count,
    })
}

#[tokio::main]
async fn main() {
    let state = Arc::new(AppState::default());
    let app = Router::new()
        .route("/stats", get(stats))
        .with_state(state);

    let listener = tokio::net::TcpListener::bind("127.0.0.1:3000").await.unwrap();
    axum::serve(listener, app).await.unwrap();
}

关键概念理解

Handler（处理器）

axum 中的 handler 是一个 async fn，接收一些 extract::Extractor 参数，返回 impl IntoResponse。处理器是请求处理链的最后一环。

1
2
3
4
5
6
7

async fn handler(
    Path(id): Path<u32>,           // 从 URL 提取
    Json(body): Json<Payload>,    // 从 body 提取 JSON
    State(state): State<Arc<MyState>>, // 共享状态
) -> impl IntoResponse {
    // ...
}

Extractor（提取器）

提取器是 axum 最优雅的设计。每个参数只要实现了 FromRequest（axum 的 trait），就可以作为处理器参数：

  • Path<T>：路径参数
  • Query<T>：查询参数
  • Json<T>：JSON 请求体
  • Header<T>：请求头
  • State<T>：共享状态
  • Request：原始请求对象

Layer（中间件层）

中间件通过 .layer() 或 .route_layer() 添加到路由。区别在于：

  • .layer()：应用于所有路由
  • .route_layer()：仅应用于特定路由

常用中间件：

  • TraceLayer：追踪请求日志
  • CorsLayer：跨域资源共享
  • RateLimitLayer：限流
  • AuthLayer：认证

IntoResponse

所有响应类型都要实现 IntoResponse。axum 内置了 &str、String、Json<T>、(StatusCode, T)、Response 等实现。

今日感悟

今天是最有成就感的一天——终于把前面积累的 async 知识变成了一个"看得见、摸得着"的 HTTP 服务器。

最大的收获是理解了 axum 的设计哲学：它不是一个什么都自己做的框架，而是站在 Tower、Tokio、Hyper 这些成熟库的肩膀上，通过组合（compose）来构建功能。Router、Handler、Extractor、Layer 这些概念都很小，但组合起来极其强大。

写代码过程中也再次体会到：async Rust 的难点从来不在语法，而在思维模式。你要习惯用"这个操作会不会阻塞？“来思考每一个步骤，习惯用 ? 传播错误，习惯用 Arc 共享状态。这需要大量练习才能形成本能。

另外，今天中间件的调试花了一些时间——特别是 TraceLayer 的输出格式。axum 的中间件是从外到内执行的，但日志打印顺序却不一定符合直觉。理解"洋葱模型"后就好多了。
