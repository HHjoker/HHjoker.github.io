核心概念总结

1. 测试金字塔 

1
2
3
4
5
6
7
8

           /\
          /  \
         /    \      端到端测试 (少量)
        /──────\
       /        \    集成测试 (适量)
      /──────────\
     /            \  单元测试 (大量)
    /──────────────\

原则：

  • 底层（单元测试）：快速、独立、覆盖广
  • 中层（集成测试）：验证模块间协作
  • 顶层（端到端）：验证完整流程

2. 测试类型对比

测试类型 位置 速度 覆盖范围 维护成本
单元测试 src/ 极快 单个函数 低
集成测试 tests/ 中等 模块间 中
文档测试 文档中 慢 示例代码 低
基准测试 benches/ 慢 性能 中

#[test]
fn test_calculator_addition() {
    // Arrange - 准备数据
    let mut calc = Calculator::new();
    let a = 5;
    let b = 3;
    
    // Act - 执行操作
    let result = calc.add(a, b);
    
    // Assert - 验证结果
    assert_eq!(result, 8);
}

#[test]
fn test_user_login() {
    // Arrange
    let db = TestDatabase::new();
    let user = User::new("test@example.com", "password123");
    db.save(&user);
    
    // Act
    let result = AuthService::login(&db, "test@example.com", "password123");
    
    // Assert
    assert!(result.is_ok());
    assert_eq!(result.unwrap().email, "test@example.com");
}

fn test_first() {
    unsafe { COUNTER += 1; }
    assert_eq!(unsafe { COUNTER }, 1);
}

#[test]
fn test_second() {
    unsafe { COUNTER += 1; }
    assert_eq!(unsafe { COUNTER }, 2);  // 依赖 test_first 先运行！
}

// ✅ 好的测试：每个测试独立
#[test]
fn test_first() {
    let mut counter = 0;
    counter += 1;
    assert_eq!(counter, 1);
}

#[test]
fn test_second() {
    let mut counter = 0;
    counter += 1;
    assert_eq!(counter, 1);  // 不依赖其他测试
}

struct UserBuilder {
    email: String,
    name: String,
    age: u32,
}

impl UserBuilder {
    fn new() -> Self {
        Self {
            email: "test@example.com".to_string(),
            name: "Test".to_string(),
            age: 30,
        }
    }
    
    fn email(mut self, email: &str) -> Self {
        self.email = email.to_string();
        self
    }
    
    fn build(self) -> User {
        User::new(&self.email, &self.name, "User", self.age)
    }
}

// 使用 trait 抽象依赖
trait Database {
    fn get_user(&self, id: u32) -> Option<User>;
    fn save_user(&self, user: &User) -> Result<()>;
}

// 生产实现
struct PostgresDatabase {
    // ...
}

impl Database for PostgresDatabase {
    // ...
}

// Mock 实现（测试用）
struct MockDatabase {
    users: std::collections::HashMap<u32, User>,
}

impl Database for MockDatabase {
    fn get_user(&self, id: u32) -> Option<User> {
        self.users.get(&id).cloned()
    }
    
    fn save_user(&self, user: &User) -> Result<()> {
        // 模拟保存
        Ok(())
    }
}

// 被测试的代码不关心具体实现
fn register_user(db: &dyn Database, user: User) -> Result<()> {
    if db.get_user(user.id).is_some() {
        return Err(Error::UserExists);
    }
    db.save_user(&user)
}

#[test]
fn test_register_new_user() {
    let db = MockDatabase::new();
    let user = create_test_user();
    assert!(register_user(&db, user).is_ok());
}

// 第 1 轮：创建空栈
// 红：写测试
#[test]
fn test_new_stack_is_empty() {
    let stack = Stack::new();
    assert!(stack.is_empty());
}

// 红：运行测试，失败（Stack 未定义）
// 绿：实现最小代码
struct Stack<T> {
    items: Vec<T>,
}

impl<T> Stack<T> {
    fn new() -> Self {
        Stack { items: Vec::new() }
    }
    
    fn is_empty(&self) -> bool {
        self.items.is_empty()
    }
}

// 绿：运行测试，通过 ✓
// 重构：代码已简洁，无需重构

// 第 2 轮：push 操作
// 红：写测试
#[test]
fn test_push_adds_item() {
    let mut stack = Stack::new();
    stack.push(1);
    stack.push(2);
    assert_eq!(stack.len(), 2);
}

// 绿：实现
impl<T> Stack<T> {
    fn push(&mut self, item: T) {
        self.items.push(item);
    }
    
    fn len(&self) -> usize {
        self.items.len()
    }
}

// 第 3 轮：pop 操作
// 红：写测试
#[test]
fn test_pop_removes_and_returns_item() {
    let mut stack = Stack::new();
    stack.push(1);
    stack.push(2);
    
    assert_eq!(stack.pop(), Some(2));
    assert_eq!(stack.pop(), Some(1));
    assert_eq!(stack.pop(), None);
}

// 绿：实现
impl<T> Stack<T> {
    fn pop(&mut self) -> Option<T> {
        self.items.pop()
    }
}

// 第 4 轮：peek 操作
// 红：写测试
#[test]
fn test_peek_returns_top_without_removing() {
    let mut stack = Stack::new();
    stack.push(1);
    stack.push(2);
    
    assert_eq!(stack.peek(), Some(&2));
    assert_eq!(stack.peek(), Some(&2));  // 仍在栈中
    assert_eq!(stack.len(), 2);
}

// 绿：实现
impl<T> Stack<T> {
    fn peek(&self) -> Option<&T> {
        self.items.last()
    }
}

// ❌ 依赖当前时间
#[test]
fn test_expiration() {
    let item = Item::new();
    assert!(item.is_expired());  // 结果取决于运行时间！
}

// ✅ 注入时间依赖
struct Item {
    created_at: DateTime,
}

impl Item {
    fn is_expired_at(&self, now: DateTime) -> bool {
        now > self.created_at + Duration::days(30)
    }
}

#[test]
fn test_expiration() {
    let item = Item::new();
    let future = item.created_at + Duration::days(31);
    assert!(item.is_expired_at(future));
}

测试检查清单

在提交代码前，问自己：

  • 是否为新增功能添加了测试？
  • 测试是否独立（不依赖执行顺序）？

  • 测试代码是否简洁易读？
  • 是否移除了调试代码（println!）？
  • 所有测试是否通过？
  • 是否运行了 clippy 和 fmt？

pub struct PasswordValidator {
    min_length: usize,
    require_uppercase: bool,
    require_lowercase: bool,
    require_digit: bool,
    require_special: bool,
}

impl PasswordValidator {
    pub fn new() -> Self {
        Self {
            min_length: 8,
            require_uppercase: true,
            require_lowercase: true,
            require_digit: true,
            require_special: false,
        }
    }
    
    pub fn validate(&self, password: &str) -> Result<(), String> {
        if password.len() < self.min_length {
            return Err(format!("Password must be at least {} characters", self.min_length));
        }
        
        if self.require_uppercase && !password.chars().any(|c| c.is_uppercase()) {
            return Err("Password must contain uppercase letter".to_string());
        }
        
        if self.require_lowercase && !password.chars().any(|c| c.is_lowercase()) {
            return Err("Password must contain lowercase letter".to_string());
        }
        
        if self.require_digit && !password.chars().any(|c| c.is_digit(10)) {
            return Err("Password must contain digit".to_string());
        }
        
        if self.require_special && !password.chars().any(|c| !c.is_alphanumeric()) {
            return Err("Password must contain special character".to_string());
        }
        
        Ok(())
    }
}

要求：

  • 测试所有验证规则
  • 测试边界条件（正好 min_length 个字符）
  • 测试组合情况
  • 使用参数化测试减少重复

// 实现 1：使用 Vec
fn find_duplicates_vec(numbers: &[i32]) -> Vec<i32> {
    let mut duplicates = Vec::new();
    for i in 0..numbers.len() {
        for j in i + 1..numbers.len() {
            if numbers[i] == numbers[j] && !duplicates.contains(&numbers[i]) {
                duplicates.push(numbers[i]);
            }
        }
    }
    duplicates
}

// 实现 2：使用 HashSet
fn find_duplicates_set(numbers: &[i32]) -> Vec<i32> {
    use std::collections::HashSet;
    let mut seen = HashSet::new();
    let mut duplicates = HashSet::new();
    
    for &num in numbers {
        if !seen.insert(num) {
            duplicates.insert(num);
        }
    }
    
    duplicates.into_iter().collect()
}

第 7 周学习感悟

这一周的测试学习让我对 Rust 有了全新的认识：

1. 测试不是负担，是安全感

以前觉得写测试很麻烦，但现在明白了：测试是代码的安全网。有了完整的测试覆盖，重构时不再提心吊胆，添加新功能时不怕破坏旧逻辑。

2. TDD 改变了我的思考方式

TDD 强迫我在写实现代码前先想清楚：

  • 这个函数的输入是什么？
  • 期望的输出是什么？
  • 边界条件有哪些？

这种"测试优先"的思维方式，让代码设计更清晰，接口更合理。

3. Rust 的测试工具链很强大

  • 内置的测试框架开箱即用
  • #[should_panic] 让异常测试变得简单
  • 文档测试确保示例代码永远正确
  • Criterion 让性能测试科学可靠

4. Mocking 是解耦的关键

通过 trait 抽象依赖，用 Mock 替换真实实现，测试变得：

  • 快速（无需真实数据库）
  • 可靠（不依赖外部服务）
  • 可重复（每次运行结果一致）

5. 测试质量 > 测试数量

100 个敷衍的测试不如 10 个精心设计的测试。好的测试：

  • 命名清晰，一眼看出测试什么
  • 独立运行，不依赖顺序
  • 覆盖边界，不只是 happy path
  • 失败时提供清晰的错误信息

6. 性能测试同样重要

功能正确只是第一步，性能达标才能真正上线。基准测试帮助我：

  • 发现意想不到的性能瓶颈
  • 验证优化是否真的有效
  • 防止性能回归

7. 测试是文档的一种形式

好的测试代码本身就是最好的文档。新同事通过阅读测试，能快速理解：

  • 这个模块的功能
  • 如何使用这些 API
  • 有哪些边界情况需要考虑
