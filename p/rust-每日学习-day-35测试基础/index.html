为什么测试很重要？

在开始今天的主题之前，让我们先理解为什么测试对于 Rust 开发如此重要：

  1. 保证代码正确性 - 测试验证代码按预期工作
  2. 防止回归 - 修改代码后，测试确保没有破坏现有功能
  3. 文档作用 - 测试展示了代码的使用方式
  4. 设计指导 - 可测试的代码往往是设计良好的代码

Rust 内置了强大的测试框架，让编写和运行测试变得非常简单。

测试的三种类型

Rust 支持三种主要测试类型：

类型 位置 用途
单元测试 与被测代码同一文件 测试单个函数/方法
集成测试 tests/ 目录 测试模块间的交互
文档测试 文档注释中 确保示例代码可运行

// src/lib.rs

/// 一个简单的计算器模块
pub mod calculator {
    /// 两数相加
    pub fn add(a: i32, b: i32) -> i32 {
        a + b
    }

    /// 两数相减
    pub fn subtract(a: i32, b: i32) -> i32 {
        a - b
    }

    /// 两数相乘
    pub fn multiply(a: i32, b: i32) -> i32 {
        a * b
    }

    /// 两数相除，返回 Result
    pub fn divide(a: i32, b: i32) -> Result<i32, String> {
        if b == 0 {
            Err("除数不能为零".to_string())
        } else {
            Ok(a / b)
        }
    }

    /// 计算阶乘
    pub fn factorial(n: u32) -> u64 {
        if n == 0 {
            1
        } else {
            n as u64 * factorial(n - 1)
        }
    }

    /// 判断是否为质数
    pub fn is_prime(n: u32) -> bool {
        if n < 2 {
            return false;
        }
        if n == 2 {
            return true;
        }
        if n % 2 == 0 {
            return false;
        }
        let sqrt_n = (n as f64).sqrt() as u32;
        for i in (3..=sqrt_n).step_by(2) {
            if n % i == 0 {
                return false;
            }
        }
        true
    }
}

#[cfg(test)]
mod tests {
    use super::calculator::*;

    // ==================== 加法测试 ====================
    #[test]
    fn test_add_positive_numbers() {
        assert_eq!(add(2, 3), 5);
        assert_eq!(add(10, 20), 30);
    }

    #[test]
    fn test_add_negative_numbers() {
        assert_eq!(add(-5, -3), -8);
        assert_eq!(add(-10, 5), -5);
    }

    #[test]
    fn test_add_zero() {
        assert_eq!(add(0, 0), 0);
        assert_eq!(add(5, 0), 5);
        assert_eq!(add(0, 5), 5);
    }

    // ==================== 减法测试 ====================
    #[test]
    fn test_subtract_positive_numbers() {
        assert_eq!(subtract(10, 3), 7);
        assert_eq!(subtract(20, 15), 5);
    }

    #[test]
    fn test_subtract_negative_result() {
        assert_eq!(subtract(3, 10), -7);
        assert_eq!(subtract(5, 20), -15);
    }

    // ==================== 乘法测试 ====================
    #[test]
    fn test_multiply_positive_numbers() {
        assert_eq!(multiply(4, 5), 20);
        assert_eq!(multiply(10, 10), 100);
    }

    #[test]
    fn test_multiply_by_zero() {
        assert_eq!(multiply(0, 5), 0);
        assert_eq!(multiply(5, 0), 0);
        assert_eq!(multiply(0, 0), 0);
    }

    #[test]
    fn test_multiply_negative_numbers() {
        assert_eq!(multiply(-3, -4), 12);
        assert_eq!(multiply(-3, 4), -12);
        assert_eq!(multiply(3, -4), -12);
    }

    // ==================== 除法测试 ====================
    #[test]
    fn test_divide_positive_numbers() {
        assert_eq!(divide(10, 2), Ok(5));
        assert_eq!(divide(20, 4), Ok(5));
    }

    #[test]
    fn test_divide_by_zero() {
        let result = divide(10, 0);
        assert!(result.is_err());
        assert_eq!(result.unwrap_err(), "除数不能为零");
    }

    #[test]
    fn test_divide_with_remainder() {
        // Rust 整数除法会截断
        assert_eq!(divide(7, 2), Ok(3));
        assert_eq!(divide(10, 3), Ok(3));
    }

    // ==================== 阶乘测试 ====================
    #[test]
    fn test_factorial_base_cases() {
        assert_eq!(factorial(0), 1);
        assert_eq!(factorial(1), 1);
    }

    #[test]
    fn test_factorial_small_numbers() {
        assert_eq!(factorial(2), 2);
        assert_eq!(factorial(3), 6);
        assert_eq!(factorial(4), 24);
        assert_eq!(factorial(5), 120);
    }

    #[test]
    fn test_factorial_larger_numbers() {
        assert_eq!(factorial(10), 3628800);
        assert_eq!(factorial(12), 479001600);
    }

    // ==================== 质数测试 ====================
    #[test]
    fn test_is_prime_small_primes() {
        assert!(is_prime(2));
        assert!(is_prime(3));
        assert!(is_prime(5));
        assert!(is_prime(7));
        assert!(is_prime(11));
    }

    #[test]
    fn test_is_prime_not_prime() {
        assert!(!is_prime(0));
        assert!(!is_prime(1));
        assert!(!is_prime(4));
        assert!(!is_prime(6));
        assert!(!is_prime(9));
        assert!(!is_prime(100));
    }

    #[test]
    fn test_is_prime_larger_primes() {
        assert!(is_prime(97));
        assert!(is_prime(101));
        assert!(is_prime(103));
    }
}

// src/lib.rs

pub mod utils {
    // 私有函数
    fn internal_helper(x: i32) -> i32 {
        x * 2
    }

    // 公开函数，内部使用私有函数
    pub fn process(value: i32) -> i32 {
        internal_helper(value) + 1
    }

    #[cfg(test)]
    mod tests {
        use super::*;

        #[test]
        fn test_internal_helper() {
            // 可以测试私有函数！
            assert_eq!(internal_helper(5), 10);
            assert_eq!(internal_helper(-3), -6);
        }

        #[test]
        fn test_process() {
            assert_eq!(process(5), 11); // (5 * 2) + 1
        }
    }
}

    let numbers = vec![1, 2, 3];
    get_item(&numbers, 10);
}

#[test]
fn test_file_operations() -> std::io::Result<()> {
    let content = std::fs::read_to_string("test.txt")?;
    assert!(!content.is_empty());
    Ok(())
}

#[test]
fn test_multiple_operations() -> Result<(), String> {
    let result1 = divide(10, 2)?;
    assert_eq!(result1, 5);
    
    let result2 = divide(20, 4)?;
    assert_eq!(result2, 5);
    
    Ok(())
}

优势：

  • 使用 ? 运算符简化错误处理
  • 不需要手动 unwrap() 或 expect()
  • 测试失败时会显示错误信息

🔑 关键概念理解

1. 测试模块的编译条件 

1
2
3
4

#[cfg(test)]
mod tests {
    // 只在 `cargo test` 时编译
    // 不会影响最终二进制文件大小
}

为什么重要？

  • 测试代码不会进入生产构建
  • 可以安全地添加大量测试而不影响性能
  • 测试可以访问私有项（因为在同一模块）

3. 测试隔离

每个测试函数都是独立的：

  • 测试之间不共享状态
  • 测试可以并行运行
  • 一个测试失败不影响其他测试

pub struct Stack<T> {

impl<T> Stack<T> {
    pub fn new() -> Self {
        Stack { items: Vec::new() }
    }

    pub fn push(&mut self, item: T) {
        self.items.push(item);
    }

    pub fn pop(&mut self) -> Option<T> {
        self.items.pop()

        self.items.last()
    }

        assert!(stack.is_empty());
        assert_eq!(stack.len(), 0);

    fn test_stack_push_and_len() {
        let mut stack = Stack::new();
        stack.push(1);
        assert_eq!(stack.len(), 1);
        
        stack.push(2);
        stack.push(3);
        assert_eq!(stack.len(), 3);
    }

    #[test]
    fn test_stack_pop() {
        let mut stack = Stack::new();
        stack.push(1);
        stack.push(2);
        stack.push(3);
        
        assert_eq!(stack.pop(), Some(3));
        assert_eq!(stack.pop(), Some(2));
        assert_eq!(stack.pop(), Some(1));
        assert_eq!(stack.pop(), None);
    }

    #[test]
    fn test_stack_peek() {
        let mut stack = Stack::new();
        assert_eq!(stack.peek(), None);
        
        stack.push(10);
        assert_eq!(stack.peek(), Some(&10));
        
        stack.push(20);
        assert_eq!(stack.peek(), Some(&20));
        
        // peek 不改变栈
        assert_eq!(stack.len(), 2);
    }

    #[test]
    fn test_stack_pop_empty() {
        let mut stack: Stack<i32> = Stack::new();
        assert_eq!(stack.pop(), None);
    }

    #[test]
    fn test_stack_with_strings() {
        let mut stack = Stack::new();
        stack.push("hello".to_string());
        stack.push("world".to_string());
        
        assert_eq!(stack.pop(), Some("world".to_string()));
        assert_eq!(stack.pop(), Some("hello".to_string()));
    }

    #[test]
    fn test_stack_lifo_order() {
        let mut stack = Stack::new();
        stack.push(1);
        stack.push(2);
        stack.push(3);
        
        // 后进先出
        assert_eq!(stack.pop(), Some(3));
        assert_eq!(stack.pop(), Some(2));
        assert_eq!(stack.pop(), Some(1));
    }

💭 今日感悟

测试是 Rust 开发中不可或缺的一部分。Rust 的测试框架设计得非常优雅：

  1. 内置支持 - 不需要额外配置，cargo test 就能运行所有测试
  2. 编译时保障 - #[cfg(test)] 确保测试代码不影响生产构建
  3. 类型安全 - 测试代码享受 Rust 的全部类型系统保护
  4. 文档即测试 - 文档测试确保示例代码永远正确

写测试的习惯应该从一开始就培养。一个好的经验法则是：每写一个函数，就为它写一个测试。这样不仅能保证代码质量，还能帮助你更好地理解代码应该如何工作。

记住：测试不是为了证明代码正确，而是为了在代码出错时能及时发现。
