学习内容

1. 所有权规则回顾

Rust 的所有权系统有三个核心规则：

  1. 每个值都有一个变量，这个变量是它的所有者
  2. 同一时间只能有一个所有者
  3. 当所有者离开作用域时，值会被丢弃

fn main() {
    // s1 是 String 的所有者
    let s1 = String::from("hello");
    
    // 所有权转移给 s2
    let s2 = s1;
    
    // s1 不再有效，这行会编译错误
    // println!("{}", s1);
    
    // s2 是现在的所有者
    println!("{}", s2); // 输出：hello
}

    let s2 = s1.clone();
    
    // 两个都可以使用
    println!("s1 = {}, s2 = {}", s1, s2);

}

fn calculate_length(s: &String) -> usize {
    s.len()
}

}

fn change(some_string: &mut String) {
    some_string.push_str(", world");
}

6. 借用规则

Rust 有严格的借用规则：

  1. 在任意给定时刻，你只能拥有以下之一：
    • 一个可变引用
    • 任意数量的不可变引用
  2. 引用必须总是有效的

fn main() {
    let mut s = String::from("hello");
    
    // 多个不可变引用 - 允许
    let r1 = &s;
    let r2 = &s;
    println!("{} and {}", r1, r2);
    
    // 可变引用 - 允许（不可变引用已不再使用）
    let r3 = &mut s;
    println!("{}", r3);
    
    // 下面的代码会编译错误：
    // let r1 = &s;
    // let r2 = &mut s; // 错误！不能同时存在可变和不可变引用
}

fn main() {
    let s = String::from("hello world");
    
    // 字符串切片
    let hello = &s[0..5];
    let world = &s[6..11];
    
    println!("{} {}", hello, world); // 输出：hello world
    
    // 省略开头或结尾
    let hello2 = &s[..5];
    let world2 = &s[6..];
    let entire = &s[..];
    
    println!("{} {} {}", hello2, world2, entire);
    
    // 切片作为参数
    let first_word = get_first_word(&s);
    println!("第一个单词：{}", first_word);
}

fn get_first_word(s: &str) -> &str {
    let bytes = s.as_bytes();
    
    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[0..i];
        }
    }
    
    &s[..]
}

关键概念理解

所有权 vs 借用

特性 所有权转移 借用（引用）
语法 let s2 = s1 let r = &s1
原变量 不再有效 仍然可用
内存 所有权转移 不获取所有权
使用场景 需要独占访问 只需要读取或临时修改

可变性规则记忆法

  • “多个读者，一个写者”：可以同时有多个不可变引用，但只能有一个可变引用
  • “读写互斥”：有可变引用时，不能有其他任何引用

切片的重要性

切片让你可以引用字符串的一部分而不需要复制，这对于性能至关重要。字符串字面量本身就是切片：&str。

今日感悟

今天深入理解了 Rust 最核心也最具挑战性的概念——所有权系统。刚开始确实觉得束缚很多，但慢慢体会到这种设计的美妙之处：

  1. 编译时保证内存安全：不需要垃圾回收器，也没有手动内存管理的风险
  2. 零成本抽象：借用和切片让高效代码变得简单
  3. 清晰的语义：代码清楚地表达了数据的流动和所有权关系

最大的收获是理解了"多个读者，一个写者"的规则。这个规则看似严格，但实际上反映了现实世界中并发安全的基本原则。

遇到的困难主要是记住何时需要 clone()，何时可以使用引用。实践中发现，多写代码、多让编译器报错，是学习所有权最好的方式。
