1. 什么是所有权（Ownership）？

所有权是 Rust 最核心的特性，它是一套内存管理规则，在编译期检查内存安全，无需垃圾回收器。

为什么需要所有权？

在其他语言中：

// JavaScript - 垃圾回收自动管理
let s1 = "hello";
let s2 = s1;  // s1 和 s2 都可以用
console.log(s1);  // ✅ "hello"
console.log(s2);  // ✅ "hello"

// C++ - 需要手动管理，容易出错
char* s1 = malloc(6);
strcpy(s1, "hello");
char* s2 = s1;  // 浅拷贝，两个指针指向同一内存
free(s1);
// free(s2);  // ❌ 双重释放！
// 或者忘记 free → 内存泄漏

// Rust - 编译期保证安全
let s1 = String::from("hello");
let s2 = s1;  // s1 的所有权转移到 s2
// println!("{}", s1);  // ❌ 编译错误！s1 已无效
println!("{}", s2);  // ✅ "hello"

2. 所有权三规则

Rust 的所有权系统基于三条简单规则：

规则 1：每个值都有一个所有者 

1
2
3
4
5

fn main() {
    let x = 5;        // x 是 5 的所有者
    let s = String::from("hello");  // s 是字符串的所有者
    let v = vec![1, 2, 3];  // v 是向量的所有者
}  // 这里所有变量都离开作用域，内存被释放

规则 2：同一时间只能有一个所有者 

1
2
3
4
5
6
7
8
9

fn main() {
    let s1 = String::from("hello");
    let s2 = s1;  // s1 的所有权转移给 s2
    
    // println!("{}", s1);  // ❌ 编译错误！
    // error[E0382]: borrow of moved value: `s1`
    
    println!("{}", s2);  // ✅ "hello"
}

规则 3：当所有者离开作用域，值被丢弃 

1
2
3
4
5
6
7
8

fn main() {
    {
        let s = String::from("hello");
        println!("{}", s);  // ✅ 可以使用
    }  // s 离开作用域，内存自动释放（调用 drop）
    
    // println!("{}", s);  // ❌ 编译错误！s 已不存在
}

3. 栈上数据 vs 堆上数据

理解所有权，首先要理解 Rust 如何存储数据。

栈（Stack）

  • 固定大小：编译时知道大小
  • 快速访问：LIFO（后进先出）
  • 自动管理：离开作用域自动弹出

fn main() {
    // 这些类型大小固定，存储在栈上
    let x: i32 = 5;
    let y: f64 = 3.14;
    let z: bool = true;
    let t: char = 'a';
    
    // 固定大小的数组也在栈上
    let arr: [i32; 5] = [1, 2, 3, 4, 5];
}

堆（Heap）

  • 动态大小：运行时才能确定大小
  • 灵活：可以增长或缩小
  • 需要管理：需要跟踪谁负责释放

    // String 大小可变，数据存储在堆上
    let s = String::from("hello");
    
    // Vec 大小可变，数据存储在堆上
    let v = vec![1, 2, 3];
    
    // Box 用于在堆上分配数据
    let b = Box::new(42);
}

4. Move 语义（所有权转移）

这是 Rust 与大多数语言最大的不同。

基础 Move

fn main() {
    let s1 = String::from("hello");
    let s2 = s1;  // Move：s1 的所有权转移给 s2
    
    // s1 现在无效了！
    // println!("{}", s1);  // ❌ 编译错误

6. Drop trait

当值离开作用域时，Rust 自动调用 drop 函数清理资源。

自动 Drop 

1
2
3
4

fn main() {
    let s = String::from("hello");
    println!("{}", s);
}  // 这里自动调用 drop(s)，释放堆内存

手动 Drop（不推荐）

struct MyResource {
    name: String,
}

impl Drop for MyResource {
    fn drop(&mut self) {
        println!("正在释放资源：{}", self.name);
        // 这里可以执行清理操作
    }
}

fn main() {
    let r1 = MyResource { name: String::from("资源 1") };
    let r2 = MyResource { name: String::from("资源 2") };
    
    println!("程序结束前");
    // 输出：程序结束前
    
}  // 自动调用 drop：
   // 正在释放资源：资源 2（后进先出）
   // 正在释放资源：资源 1

// 创建字符串
fn create_string() -> String {
    String::from("hello")
}

// 获取所有权
fn take_string(s: String) {
    println!("收到：{}", s);
}  // s 被 drop

// 借用（不获取所有权）- 明天学习
fn borrow_string(s: &String) {
    println!("借用：{}", s);
}

// 返回部分所有权
fn return_part(s: String) -> (String, String) {
    let prefix = s[..2].to_string();
    (s, prefix)
}

fn main() {
    // 创建
    let s1 = create_string();
    
    // 转移所有权
    take_string(s1);
    // println!("{}", s1);  // ❌ s1 已无效
    
    // 重新创建
    let s2 = String::from("world");
    
    // 返回多个值
    let (s3, prefix) = return_part(s2);
    println!("原字符串：{}, 前缀：{}", s3, prefix);
}

🔑 关键概念理解

1. Move 是 Rust 的核心 

1

fn main() {
    let s1 = String::from("hello");
    let s2 = s1;  // Move 发生
    
    // 为什么 Rust 要这样设计？
    // 答案：防止双重释放（double free）
}

如果 Rust 允许 s1 和 s2 都指向同一块堆内存：

1
2
3
4

危险情况：
s1 和 s2 都指向同一块内存
当 s1 离开作用域 → 释放内存
当 s2 离开作用域 → 再次释放同一内存 → ❌ 双重释放！

Rust 通过 Move 语义，确保只有一个变量负责释放内存。

2. 浅拷贝 vs 深拷贝 

1

// 浅拷贝（Move）

// 深拷贝（Clone）
let s3 = String::from("world");
let s4 = s3.clone();  // 复制所有数据到新的堆内存

// 5. Drop 顺序
struct NamedDrop(&'static str);

impl Drop for NamedDrop {
    fn drop(&mut self) {
        println!("Dropping {}", self.0);
    }
}

fn main() {
    let a = NamedDrop("a");
    let b = NamedDrop("b");
    let c = NamedDrop("c");
}
// 输出：
// Dropping c
// Dropping b
// Dropping a

Day 8: 引用与借用（References and Borrowing）

明天我们将学习如何在不转移所有权的情况下使用数据——引用和借用。这是 Rust 最强大的特性之一，让你既能保证安全，又能灵活地共享数据。

💭 今日感悟

所有权是 Rust 的灵魂。今天的学习让我深刻理解到：

  1. 安全不是免费的，但 Rust 让它零成本：编译期检查，运行时无开销
  2. Move 语义初看限制，实为解放：不再担心内存泄漏和双重释放
  3. 显式优于隐式：clone() 明确告诉读者"这里发生了复制"

刚开始可能会觉得所有权规则很繁琐，但这是 Rust 给你的一份礼物——用编译期的"麻烦"换取运行时的安心。

最让我印象深刻的是 Drop trait 的设计。它让资源管理变得如此优雅，不再需要 try-finally 或 using 块，作用域结束自动清理。
