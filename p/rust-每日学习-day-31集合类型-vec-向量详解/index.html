什么是 Vec？

Vec（Vector 的缩写）是 Rust 中最常用的集合类型，它是一个可增长的数组。与我们之前学过的数组不同，Vec 的长度可以在运行时动态变化。

1
2
3
4
5

// 数组 - 长度固定
let arr: [i32; 3] = [1, 2, 3];

// Vec - 长度可变
let vec: Vec<i32> = vec![1, 2, 3];

fn main() {
    let v = vec![1, 2, 3, 4, 5];
    
    // 方法 1: 使用索引（可能 panic）
    let third = &v[2];
    println!("第三个元素：{}", third);
    
    // 方法 2: 使用 get（返回 Option）
    match v.get(2) {
        Some(third) => println!("第三个元素：{}", third),
        None => println!("索引越界"),
    }
    
    // 安全访问 vs 不安全访问
    let out_of_bounds = v.get(10); // None，安全
    // let panic_now = &v[10];     // 这会 panic！
    
    println!("越界访问结果：{:?}", out_of_bounds);
}

fn main() {
    let mut v = vec![1, 2, 3];
    
    // 修改元素
    v[0] = 10;
    println!("修改后：{:?}", v); // [10, 2, 3]
    
    // 添加元素到末尾
    v.push(4);
    println!("push 后：{:?}", v); // [10, 2, 3, 4]
    
    // 删除最后一个元素并返回
    let last = v.pop();
    println!("pop 返回：{:?}", last); // Some(4)
    println!("pop 后：{:?}", v); // [10, 2, 3]
    
    // 在指定位置插入
    v.insert(1, 20);
    println!("insert 后：{:?}", v); // [10, 20, 2, 3]
    
    // 删除指定位置的元素
    let removed = v.remove(1);
    println!("remove 返回：{}", removed); // 20
    println!("remove 后：{:?}", v); // [10, 2, 3]
}

fn main() {
    let mut v = vec![1, 2, 3, 4, 5];
    
    // 获取长度
    println!("长度：{}", v.len()); // 5
    
    // 检查是否为空
    println!("是否为空：{}", v.is_empty()); // false
    
    // 获取容量（分配的内存空间）
    println!("容量：{}", v.capacity());
    
    // 预留容量
    v.reserve(10);
    println!("预留后容量：{}", v.capacity());
    
    // 截断到指定长度
    v.truncate(3);
    println!("截断后：{:?}", v); // [1, 2, 3]
    
    // 清空所有元素
    v.clear();
    println!("清空后：{:?}", v); // []
    println!("清空后长度：{}", v.len()); // 0
}

fn main() {
    let v = vec![1, 2, 3, 4, 5];
    
    // 方法 1: 索引遍历
    println!("索引遍历:");
    for i in 0..v.len() {
        println!("v[{}] = {}", i, v[i]);
    }
    
    // 方法 2: 迭代器（推荐）
    println!("\n迭代器遍历:");
    for element in &v {
        println!("element = {}", element);
    }
    
    // 方法 3: 可变迭代器
    let mut v2 = vec![1, 2, 3];
    for element in &mut v2 {
        *element += 10;
    }
    println!("\n修改后：{:?}", v2); // [11, 12, 13]
    
    // 方法 4: 带索引的迭代器
    println!("\n带索引遍历:");
    for (index, value) in v.iter().enumerate() {
        println!("v[{}] = {}", index, value);
    }
}

🔑 关键概念理解

1. Vec vs 数组

特性 数组 Vec
长度 编译时固定 运行时可变
类型 [T; N] Vec<T>
内存 栈上（通常） 堆上
性能 略快 有动态分配开销

2. 容量（Capacity）vs 长度（Length）

  • 长度：Vec 中实际存储的元素数量
  • 容量：Vec 已分配的内存空间能容纳的元素数量

当长度超过容量时，Vec 会自动重新分配更大的内存空间（通常是 2 倍），并将原有元素复制过去。

3. 为什么索引访问可能 panic？ 

1
2

let v = vec![1, 2, 3];
let x = v[10]; // 如果索引越界，直接 panic！

这是 Rust 的安全机制。使用 get() 方法更安全：

1
2
3
4

match v.get(10) {
    Some(value) => println!("值：{}", value),
    None => println!("索引越界"),
}

4. 迭代器借用规则

  • &v 或 v.iter()：不可变借用，读取元素
  • &mut v 或 v.iter_mut()：可变借用，修改元素
  • v：转移所有权，消耗 Vec

fn calculate_average(numbers: Vec<i32>) -> f64 {
    if numbers.is_empty() {
        return 0.0;
    }
    
    let sum: i32 = numbers.iter().sum();
    sum as f64 / numbers.len() as f64
}

fn main() {
    let nums = vec![1, 2, 3, 4, 5];
    let avg = calculate_average(nums);
    println!("平均值：{}", avg); // 3.0
}

fn remove_duplicates(mut v: Vec<i32>) -> Vec<i32> {
    let mut result = Vec::new();
    
    for num in v {
        if !result.contains(&num) {
            result.push(num);
        }
    }
    
    result
}

fn main() {
    let nums = vec![1, 2, 2, 3, 1, 4, 3, 5];
    let unique = remove_duplicates(nums);
    println!("{:?}", unique); // [1, 2, 3, 4, 5]
}

fn two_sum(nums: Vec<i32>, target: i32) -> Option<(usize, usize)> {
    for (i, &a) in nums.iter().enumerate() {
        for (j, &b) in nums.iter().enumerate() {
            if i != j && a + b == target {
                return Some((i, j));
            }
        }
    }
    None
}

fn main() {
    let nums = vec![2, 7, 11, 15];
    let target = 9;
    
    match two_sum(nums, target) {
        Some((i, j)) => println!("找到：索引 {} 和 {}", i, j),
        None => println!("未找到"),
    }
}

💭 今日感悟

今天深入学习了 Vec，它是 Rust 中最常用的集合类型。有几个关键点让我印象深刻：

  1. 安全性优先：Rust 在索引访问时选择 panic 而不是返回未定义行为，这体现了 Rust 的安全理念。使用 get() 方法可以优雅地处理越界情况。

  2. 容量与长度的区别：理解这个概念对于性能优化很重要。如果预先知道 Vec 的大小，使用 Vec::with_capacity() 可以避免多次内存重新分配。

  3. 借用与迭代的配合：Rust 的借用检查器确保我们在遍历 Vec 时不会意外修改它，这避免了很多潜在的 bug。

  4. 从数组到 Vec 的思维转变：从固定长度到动态长度，这是一个重要的思维转变。在实际项目中，Vec 的使用频率远高于数组。
